# L4 多设备采集方案 Spec

状态：已整理为产品/交互规格  
日期：2026-07-04  
适用范围：Lab Data 检测员侧移动端 L4 采集详情页  
参考原型：`src/screens/Collect.jsx`、`src/screens/CollectStructured.jsx`、`src/mock.js`

## 1. 背景

Lab Data 的 L4 采集详情页承接检测员现场采集动作。现有单设备 L4 已支持基础信息、设备信息、环境信息、试验字段录入、采集方式自适应、上传与流程锁定。

实际业务中存在一个试验项由多个试验子项组成的情况，例如「结构尺寸检查-导体&绝缘厚度&金属屏蔽」。该试验项在 LIMS/Web 后台是一个检测项，但现场采集需要多台设备共同完成：

| 试验子项 | 典型设备 | 采集方式 | 数据形态 |
| --- | --- | --- | --- |
| 导体 | 绝缘层机械性能智能测试系统-无人化版 | 设备直连 | 红/黄/绿相别，各相结构、根数、直径 |
| 绝缘厚度测量 | 台式测厚仪 | 手工录入 | 红/黄/绿相别，每相多个厚度测量点 |
| 金属屏蔽 | 数显卡尺 | 蓝牙采集 | 红/黄/绿相别，铜带厚度、宽度、搭盖宽度 |

如果仍按「一个 L4 页面 = 一台设备 = 一组字段」处理，检测员需要在多个页面之间跳转，容易漏采、错选设备或把读数上传到错误设备配置。L4 需要支持一个试验项下的多设备协同采集。

## 2. 目标

1. 在一个 L4 页面内完成同一试验项下多个试验子项的数据采集。
2. 明确绑定「试验子项 - 设备 - 采集方式 - 字段」关系，避免读数串设备。
3. 让检测员按子项、相别、次数逐步完成采集，并能看到整体完成度。
4. 保留现有采集方式差异：设备直连、拍照识别、蓝牙、外部程序（含串口通路）、手工。
5. 结论由 LIMS 按判定配置回显，移动端不手工录入结论。
6. 与 LIMS 流程节点一致：进入审核后锁定；退回试验检测后可重新采集并重新上传。

## 3. 非目标

1. 不在本规格中重做 L1/L2/L3 信息架构。
2. 不调整 Web 后台的完整配置流程，只约定移动端必须拿到的配置结构。
3. 不在移动端实现检测标准、阈值、结论算法。
4. 不把多设备方案扩展成任意自由组合。设备选择必须受后台配置约束。

## 4. 推荐方案

采用「复合试验项 L4」方案：L4 仍以一个试验项为页面上下文；当该试验项包含 `subs` 试验子项时，进入多设备采集模式。页面主工作单元是试验子项，设备信息随当前子项联动展示。

### 4.1 备选方案对比

| 方案 | 优点 | 风险 | 结论 |
| --- | --- | --- | --- |
| 拆成多个独立 L4 页面 | 实现简单，沿用单设备逻辑 | 检测员频繁跳转，整体进度割裂，容易漏采 | 不采用 |
| 一个页面内自由切换设备和字段 | 灵活，适合演示 | 设备与字段可能被误配，数据正确性风险高 | 不采用 |
| 复合试验项：子项驱动设备与字段 | 与 LIMS 试验项一致，关系清晰，便于审核追溯 | 需要后端提供子项配置 | 采用 |

### 4.2 核心原则

1. 试验项是 L4 页面入口，试验子项是采集工作单元。
2. 当前子项决定当前设备、采集方式、字段模板、相别/次数模型。
3. 如果一个子项允许多台候选设备，设备切换只能发生在该子项的候选设备集合内，并记录实际采集设备。
4. 每条读数必须带上 `testItemId`、`subItemId`、`deviceId`、`phase`、`repeatIndex`、`fieldKey`、`source`。
5. 设备直连与外部程序（含串口）由上位机/平板程序写库后 App 自动展示；蓝牙、拍照识别、手工按相别/次数逐条完成。原型可用「一键采集」演示回填，生产无此按钮。
6. 上传可以按单元格单独上传，也可以在底部执行上传全部。

## 5. 页面结构

### 5.1 顶部区域

顶部保持现有 L4 框架：

1. AppBar：标题为「检测任务」，支持返回。
2. 状态印章：未检测、检测中、已检测。退回复测场景下，检测员刚进入且尚未修改/重置数据时不展示印章（保留历史上传数据供核对）；一旦修改或重置，按填写与上传进度展示印章。
3. 流程提示：若数据已进入组内审核及以后节点，显示锁定状态；若从审核退回，显示退回原因。
4. 基础信息：样品编号、样品名称、试验名称、试验次数。

试验次数展示逻辑：

1. 普通试验项：展示总次数，例如 `15 次`。
2. 复合试验项：展示主次数，并补充「含 N 个试验子项 · 试验次数随任务下发，不可修改」。
3. App 端一律不可修改试验次数；按对接模式同步来源不同：
   - 轻量版 LIMS+数采：LIMS 可初始化试验项次数；设备直采可在数采 Web「设备采集配置」改次数，App L4 同步。
   - 物资版 LIMS+数采：电缆样品次数=样段×试样×芯数，非电缆=样段×试样；设备直采不可在「设备采集配置」改次数，App 同步 LIMS 样品数据。
   - 第三方 LIMS+数采：设备直采可在「设备采集配置」改次数，App L4 同步。

### 5.2 设备信息

设备信息区必须突出当前子项对应的设备：

1. 显示采集方式徽章。
2. 显示检测设备、设备编号、设备型号、所属工位（未绑定则展示「未绑定工位」）。
3. 若当前子项存在候选设备，展示候选设备切换控件；切换后只影响当前子项。
4. 若当前试验项包含多个子项，提示「该试验项含多个试验子项，当前设备随子项联动」。

禁止在该区域提供不受配置约束的全局设备切换。

### 5.3 试验子项切换

复合试验项展示「试验子项」切换控件，推荐使用 SegmentedSwitch 或横向标签：

1. 标签显示子项名称，例如「导体」「绝缘厚度测量」「金属屏蔽」。
2. 每个子项显示状态摘要：待采集、待上传、已上传。
3. 默认选中第一个未完成子项；若全部完成，默认选中第一个子项。
4. 切换子项时，保留当前子项已填数据，不清空。

### 5.4 试验数据

试验数据区采用主从结构：

1. 左侧为相别/次数列表。
2. 右侧为当前相别/次数的字段录入区。
3. 含相别的子项按红、黄、绿展示；每相可有 1 次或多次。
4. 不含相别的子项按第 1 次、第 2 次展示。
5. 多点测量字段使用多个输入框展示，例如「绝缘各测量点厚度」显示 6 个点位输入。

每个相别/次数单元的状态：

| 状态 | 展示 | 说明 |
| --- | --- | --- |
| 待采集 | 黄色提示图标 | 该单元还没有有效读数 |
| 待上传 | 已采集但未上传 | 可执行确认并上传本相/本次 |
| 已上传 | 绿色成功图标 | 数据已上传并归档 |
| 已锁定 | 锁图标或只读态 | 流程进入审核后禁止修改 |
| 上传失败 | 错误提示 | 保留读数，可重试上传 |

### 5.5 结论

结论区按后台判定结果回显：

1. 含相别子项：每个相别展示一个结论。
2. 无相别子项：展示一个结论。
3. 结论字段只读，文案为「本相上传后回显」或「上传后回显」。
4. 若子项内有多次/多点测量，同一相别的测量数据合并判定为一个结论。

### 5.6 底部操作

底部操作保持常驻：

1. `重置全部`：清空当前 L4 页面未上传数据；已锁定数据不可清空。
2. `上传全部（x/y）`：上传所有已采集但未上传的单元。
3. `完成并退出`：当全部上传完成后返回上一层。

若只完成部分相别/次数，允许单元级上传，L4 状态进入检测中。

## 6. 采集方式行为

| 采集方式 | 入口 | 读写规则 | 附件 | 典型文案 |
| --- | --- | --- | --- | --- |
| 设备直连 auto | 生产：自动回填（无按钮）；原型可演示「一键采集」 | 上位机整批写库后 App 只读展示；不可手输 | 无需附件 | 设备直连 · 上位机算毕整批写库 |
| 拍照识别 ocr | 拍照识别 | 识别规则验证通过才显示拍照入口；识别结果默认锁定，可编辑校正；未通过验证回退手工录入 | 每次保留一张识别参照图 | 逐条拍摄仪器读数屏自动识别 |
| 蓝牙 ble | 连接采集 | 逐条同步，可手工修正 | 可拍照/上传参照图 | 蓝牙数显卡尺 · 逐相连接同步读数 |
| 外部程序 external（含串口通路） | 生产：无采集按钮，可「查看已采」；原型对串口通路可演示「一键采集」 | 工业平板外部采集程序代采写库（串口是外部程序的一种通讯通路，不单独成一类业务方式）；App 展示已采数据，异常可手输兜底；已上传态后不可再改 | 可按业务需要上传附件 | 外部程序代采写库（含串口） |
| 手工 manual | 录入本相读数 | 检测员手工填写 | 可按业务需要上传附件 | 单机设备 · 读数手工录入 |

说明：原型中设备直连与外部程序/串口通路上的「一键采集」仅用于演示整批回填，**生产环境不要该按钮**。

拍照识别只有在识别规则验证状态为已通过时出现；未通过时回退手工录入，并给出明确提示。

### 7.2 试验项配置

```ts
type TestItemConfig = {
  id: string;
  name: string;
  method?: CollectMethod;

  tpl?: string;
  phased?: boolean;

  node: string;

```

### 7.3 采集数据

```ts
type CollectCell = {
  testItemId: string;
  subItemId?: string;
  deviceId: string;
  phase?: '红' | '黄' | '绿';
  repeatIndex: number;

  fieldKey: string;
  label: string;
  value: string | number | string[] | number[];

};

  id: string;
  kind: 'photo' | 'upload' | 'ocr-source';
  url?: string;
};
```

### 7.4 必填追溯字段

每次上传必须包含：

1. 任务 ID、样品 ID、试验项 ID。
2. 试验子项 ID；普通 L4 可为空。
3. 实际采集设备 ID。
4. 采集方式。
5. 相别与次数索引。
6. 字段 key、字段值、单位。
7. 操作人、采集时间、上传时间。
8. 附件 ID 与 OCR 原图 ID。
9. 原始设备返回报文或原始识别结果；没有则为空。

## 8. 状态汇总

### 8.1 单元状态

单元指一个子项下的一个相别/次数。

1. `idle`：未采集。
2. `filled`：已采集或已录入，未上传。
3. `uploading`：上传中。
4. `uploaded`：已上传。
5. `failed`：上传失败，可重试。
6. `locked`：流程锁定，不可改；也可以作为页面级只读态叠加在 `uploaded` 数据上。

### 8.2 子项状态

1. 待采集：所有单元均为 `idle`。
2. 采集中：存在 `filled`、`uploaded` 或 `failed`，但未全部 `uploaded`。
3. 已上传：该子项全部单元为 `uploaded`。

### 8.3 L4 状态印章

1. 未检测：没有任何单元已上传。
2. 检测中：至少一个单元已上传，但未全部上传。
3. 已检测：全部必采单元已上传。

## 9. 异常与边界

1. 缺少设备采集方式：L4 禁止开始采集，提示后台维护设备采集方式。
2. 缺少子项设备：该子项显示配置异常，不影响其他子项查看。
3. 设备直连无数据：保留待采集状态，提示「暂未取到设备数据」并允许重试。
4. 蓝牙连接失败：保留当前输入，允许重新连接或手工录入。
5. OCR 规则未通过：隐藏拍照识别入口，回退手工录入。
6. 上传失败：读数与附件留在本地页面状态中，按钮变为重试上传。
7. 流程锁定：字段、附件、重置、上传入口禁用，只展示历史数据。
8. 审核退回：数据解锁，修改任一字段后该单元上传状态变为待上传。
9. 候选设备切换：只允许在当前子项候选设备内切换；切换后已采数据必须标明原采集设备，不得静默覆盖。

## 10. 接口约定

### 10.1 读取 L4 配置

移动端进入 L4 时读取完整配置：

```http
GET /api/mobile/collect-context?taskId=...&sampleId=...&testItemId=...
```

响应必须包含：

1. 基础样品与任务信息。
2. 当前流程节点与是否锁定。
3. 试验项配置。
4. 试验子项配置与设备配置。
5. 字段模板、单位、是否必填、多点测量数量。
6. 已采集/已上传历史数据。
7. 附件与 OCR 参照图信息。
8. 试验项级检测起止时间 `timing`（`startedAt` / `endedAt` / `startedBy`）。

```json
{
  "timing": {
    "startedAt": "2026-07-20T14:30:05+08:00",
    "endedAt": null,
    "startedBy": "operator-001"
  }
}
```

### 10.1.1 记录试验开始时间

```http
POST /api/mobile/collect/test-timing/start
```

请求体：`taskId`、`sampleId`、`testItemId`；可选 `startedAt`（默认服务端时间）。

- 无已上传数据时允许首次写入或覆盖 `startedAt`。
- 已有上传数据时拒绝覆盖（409）。

### 10.1.2 记录试验结束时间

推荐在 `POST /api/mobile/collect/upload` 处理最后一次单元上传时，若该试验项全部 uploaded，服务端自动写入 `endedAt`。客户端无需提供结束时间按钮。

### 10.2 拉取设备数据

```http
POST /api/mobile/collect/read
```

请求包含 `taskId`、`sampleId`、`testItemId`、`subItemId`、`deviceId`、`phase`、`repeatIndex`、`method`。

设备直连可以返回整批单元；蓝牙、OCR、手工通常返回单个单元。

### 10.3 上传采集数据

```http
POST /api/mobile/collect/upload
```

上传单位可以是单个单元或多个单元。服务端必须按 `subItemId + deviceId + phase + repeatIndex + fieldKey` 幂等写入，避免重复上传产生重复数据。

## 11. 验收标准

1. 进入包含 `subs` 的试验项时，L4 自动使用多设备采集模式。
2. 页面能展示试验子项切换、当前设备信息、采集方式徽章与子项完成状态。
3. 切换子项时，设备、字段、相别/次数列表随子项变化，已采数据不丢失。
4. 导体子项可整批采集，字段只读。
5. 绝缘厚度测量可按红/黄/绿相别手工录入多个测量点。
6. 金属屏蔽可按相别蓝牙采集，并允许上传参照图。
7. 单个相别/次数上传后显示已上传；全部上传后 L4 印章为已检测。
8. 结论只读，并在对应数据上传后回显。
9. 流程进入组内审核及以后节点时，所有字段、附件与上传操作锁定。
10. 审核退回到试验检测时，可修改并重新上传，页面展示退回原因。
11. 上传数据中可以追溯到实际子项、设备、相别、次数、字段与采集方式。

## 12. 原型落点

当前代码中已有可参考实现：

1. `src/screens/Collect.jsx`：普通 L4 与采集方式自适应基础逻辑。
2. `src/screens/CollectStructured.jsx`：复合试验项、多子项、多设备、多相别采集原型。
3. `src/mock.js`：结构尺寸检查示例数据、子项字段、设备与采集方式配置。

后续实现时建议将 `CollectStructured` 收敛为生产语义：

1. 子项驱动设备与字段，避免全局自由切设备。
2. 将演示态 `cells`、`attach`、`busy` 抽象为可与后端同步的采集单元状态。
3. 将字段模板、相别、次数、结论从 mock 迁移到后端配置。
4. 补充上传失败、配置异常、OCR 未验证、流程锁定等真实状态。

## 13. 实施注意

1. 后台设备管理必须维护采集方式，否则移动端无法判断采集入口。
2. 设备采集配置必须能表达一个试验项下多个试验子项。
3. LIMS 结果判定必须支持按子项、相别聚合后回显结论。
4. 移动端不能仅凭设备名称匹配字段，必须使用稳定 ID。
5. 附件与 OCR 原图必须按单元归档，不能只挂在整个试验项上。
6. 本地临时状态要防止误清空，尤其是蓝牙、OCR、手工录入场景。
