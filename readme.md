# 数蚕 Lab Data 设计系统

实验室数智化系统（**Lab Data**）的品牌与界面设计系统。沉淀颜色、字体、状态徽章体系、可复用组件，以及**检测员侧移动端 UI Kit**，供后续 app/web 设计与原型快速、统一地产出。

> 厂商：杭州数蚕智能科技有限公司 · 产品：Lab Data 实验室数智化系统 · 当前版本 V2.0 · 域名 shucantech.cn

## 工程运行

当前目录已整理为标准 Vite + React 项目。后续迭代主要从 `src/App.jsx`、`src/screens/`、`components/` 和 `tokens/` 开始。

```bash
npm install
npm run dev
npm run build
```

- `src/screens/`：由检测员移动端 UI Kit 转成的 React 页面模块
- `src/design-system.js`：统一导出设计系统组件
- `styles.css` / `tokens/`：全局设计 token 入口与变量
- `ui_kits/app/index.html`：旧版静态预览保留用于对照

## 一、产品背景

Lab Data 解决一件事：**把试验设备的数据实时、自动地采集，并自动上传到业务管理系统（LIMS）**，替代"手摇表、人记录、U 盘倒数据"的作坊式流程，杜绝人工改数，提升试验室运行效率。典型用户是**电力物资检测中心**（绝缘工具、电缆、脚扣、安全帽等安全工器具的力学/电气检测）。

### 两个角色
- **检测员**：在检测工位现场做试验、采集并上传数据。移动端（平板）使用。
- **管理员 admin**：查看所有检测员工作情况、所有样品/任务的检测进度与数据。

### 两个产品面
1. **移动端 app（本设计系统的重点）**：检测员现场采集工具，平板竖屏。本期重构检测员全流程，admin 侧二期。
2. **Web 管理后台（已有）**：`实验室数智化系统`，给系统管理员/实施做配置——设备、采集表、第三方试验表、设备采集配置、数据映射、图像识别规则、任务与日志。深蓝侧边栏 + Element 风格表格（RuoYi 体系）。**本设计系统不照搬 web 视觉**，app 走更轻的浅色卡片风。

### 核心数据模型（来自 web 后台，设计务必遵循）
- **设备**：含「所在区域」字段（如*力学检测室2 / 无人化试验室*）——即**工位/区域**。一台设备绑定其可做的采集配置。**应在 Web「设备管理」为每台设备维护一个「采集方式 / 连接方式」字段**（设备直连 / 蓝牙 / 拍照识别 / 手工），app 据此渲染采集方式徽章、决定采集页交互（能否一键采集、能否手输）；缺这个字段，「蓝牙同步」等标识无从判断。建议同时维护「是否绑定工位」（决定是否进入「非工位设备」入口）。
- **设备采集配置**：`试验设备 → 采集表 → 第三方试验表 → 试验次数 → 试验字段` 绑定关系。**采集动作绑定到具体某台设备**（选错设备＝采到别的仪器的数，这是数据正确性命门）。
- **多对多**：一台设备可做多个试验项；一个试验项也可被多台设备做。
- **试验字段**：如 `cd 长度 / zxdz 正向电阻 / fxdz 反向电阻 / pjz 平均值 / r20 20℃电阻 / wd 温度 / sd 湿度 / bcz 标称值 / syz 试验值 / jl 结论`。
- **散落设备**（不挂工位系统）分三类，靠**采集方式**兜底：①老旧无网设备 → 📷拍照识别(OCR)/手工；②便携设备 → 🔵蓝牙扫码同步；③辅助单机（卡尺、测温枪）→ ✏️手工录入。联网设备走 ⚡自动采集。

## 二、信息架构（检测员侧 · 本期）

```
登录
└ 首页（轮播/通知 · 快捷入口[待检/检测/已检] · 检测模块入口 · 个人检测统计图）
   └ 检测模块
      ├ 顶部常驻：当前工位上下文（可切换，记住上次）
      ├ 右上 Switch：按设备 ┃ 按样品
      ├ 「便携设备 · 扫码连接」快捷入口
      ├ 按设备：设备列表 → 选设备 → 选样品 + 该设备可做的试验项 → 采集详情
      └ 按样品：样品列表 → 选样品 → 该样品全部试验项(标注所需设备/采集方式) → 采集详情
   └ 采集详情（L4）：基础信息 · 设备信息 · 环境信息 · 试验字段录入
      采集方式自适应：⚡自动 / 📷拍照识别 / 🔵蓝牙 / ✏️手工 → 上传
   └ 我的（权限、修改密码、退出登录）
```

旧设计层级 L1 首页 / L2 任务列表(仅 admin) / L3 样品详情 / L4 检测详情，本期在检测员侧重构并新增「工位/设备/样品」入口。

## 三、内容规范 CONTENT FUNDAMENTALS

- **语言**：简体中文为主，编号/字段名/型号保留英文与拼音缩写（如 `YM-WRH-05`、`dldl_zldz`、`SC2025/0293`）。
- **人称**：界面以**第二人称隐含/无人称祈使**为主（"请输入试验编号"、"请选择第三方试验表"），个人页用"我的"。不用"您"，不用第一人称。
- **语气**：专业、克制、工程化。陈述事实与状态，不卖萌、不营销腔。功能即标签（"下发""采集""上传""重置""查询""新增""导入"），动词单刀直入。
- **大小写/排版**：中文不加句号于短标签；英文状态副标可全大写加宽（登录页 `WELCOME`）。数字、编号、时间统一半角。
- **时间格式**：`2025-08-27 13:18:10`（24 小时、补零、空格分隔）。
- **状态词固定**：未检测 / 检测中 / 已完成 / 已检测 / 已逾期 / 未上传 / 已上传 / 已采集 / 成功 / 失败。**不要近义替换**。
- **emoji**：不用于正式界面文案。本系统文档里出现的 ⚡📷🔵✏️ 仅作**采集方式示意图标**，实际界面用图标组件而非彩色 emoji。
- **示例文案**：
  - 空状态搜索框："请输入试验编号、试验名称进行搜索"
  - 任务卡："任务编号 SC2025/0293" · "委托单位 杭州数蚕智能科技有限公司" · "下发时间 2025-08-27 13:18:10"
  - 按钮组：`重置` `查询` / `取消` `确定` / `重置` `上传`

## 四、视觉基础 VISUAL FOUNDATIONS

- **主色**：品牌深蓝 `--blue-700 #1D3F8C`（取自 Logo）；交互/操作蓝 `--blue-500 #2563EB`（按钮、链接、选中）。蓝是唯一强调色，克制使用。
- **背景**：页面浅灰 `--bg-app #F4F6F9`，内容为**白色卡片**。**无渐变背景、无大面积色块、无紫蓝渐变**。首页顶部可放一条轻品牌色装饰条/标题左侧 4px 蓝色竖条（沿用旧稿）。
- **字体**：CJK 走系统字体栈（PingFang SC / 鸿蒙 / 微软雅黑 / Noto Sans SC，详见字体方案章）。仪表读数用 `tabular-nums` 等宽数字保证小数点对齐。
- **间距**：4px 基准栅格；页面左右安全边距 20px；卡片内边距 16px；区块间距 24px。
- **圆角**：卡片 12px、按钮/输入 8px、徽章 6px、状态标签胶囊 999px。统一中等圆角，不夸张。
- **阴影**：卡片走"**轻阴影 + 极淡描边**"（`--shadow-card` + `1px var(--border-default)`）。**不用重投影**、**不用彩色左边框**、**不用霓虹光晕**。底部 Tab 栏用上向轻阴影。
- **描边**：常规 1px `#E5E8EE`；输入聚焦 1px 蓝 + 3px 蓝色 18% 外发光。
- **动效**：以**淡入 + 轻移**为主，缓动 `ease-out` 200ms。**无弹跳、无装饰性循环动画**。
- **Hover（平板以触控为主，hover 仅 web/调试）**：背景转 `--surface-hover #EEF1F5`，或主色加深一档。
- **按压 Press**：轻微缩放 `scale(0.97)` + 颜色加深一档（主按钮 `--blue-500 → --blue-600`），无弹跳。
- **透明/模糊**：仅用于弹窗遮罩（黑 ~45%）与底部 Tab 栏可选毛玻璃；正文区域不滥用。
- **图像色调**：实拍图（设备/仪表）偏冷、写实、无滤镜，不做暖调或黑白处理。
- **卡片定义**：白底 + 12px 圆角 + 1px `#E5E8EE` 描边 + `--shadow-card` 轻阴影。选中态加蓝色 1.5px 描边 + `--surface-selected #EEF4FF` 浅蓝底。

## 五、图标规范 ICONOGRAPHY

- **图标库**：旧 app 与 web 未内置专有图标字体；web 后台为 Element/RuoYi 体系（element-plus 图标 + 部分 SVG）。本设计系统统一采用 **Lucide**（线性、2px 描边，开源、CDN 可用），与"工程化、克制"的气质一致。**这是替代选择，非原系统图标，已在此标注**——若你们有内部图标资产，请提供，我替换。
- **引入方式**：CDN `lucide`，`<script src="https://unpkg.com/lucide@latest"></script>` 后 `lucide.createIcons()`；或用 `lucide-react`。组件里图标尺寸默认 20px，描边色随文字色。
- **采集方式图标**（语义固定，组件内用 Lucide 对应）：⚡自动→`zap`；📷拍照识别→`scan-line`/`camera`；🔵蓝牙→`bluetooth`；✏️手工→`pencil`。
- **状态图标**：上传成功→`check-circle`(绿)；未上传→`alert-circle`(橙)；试验状态用**纯色圆点**（非图标），见状态徽章体系。
- **emoji / unicode**：不作为界面图标使用。文档示意除外。
- **Logo**：圆形深蓝「S」标，见 `assets/logo-mark.png`。横版组合为 `S 标 + "Lab Data"`。

## 六、状态徽章体系（核心规范，原样沉淀自客户「状态分类」表）

| 维度 | 取值与样式 |
|---|---|
| **任务状态** | 未检测 / 检测中 / 已完成 / 已逾期 |
| **样品检测状态**（胶囊标签 Tag） | 未检测=印章黄 · 检测中=浅蓝 · 已完成=绿 · 已逾期=红 |
| **试验状态**（实心圆点 Dot） | 未检测=印章水印黄填充 `--status-pending-dot` · 检测中=浅蓝 · 已检测=绿 |
| **上传状态**（图标） | 未上传=橙⚠ `alert-circle` · 已上传=绿✓ `check-circle` |

色值见 `tokens/colors.css` 的 `--status-*` / `--upload-*`。组件实现见 `components/`（StatusTag / StatusDot / UploadStatus）。
